Demonstração: SQL Injection (Vulnerável) vs Prepared Statements (Seguro)

⚠️ Sistema Vulnerável

SEM Prepared Statements
🧪 Teste o ataque SQL Injection: Usuário: ' OR '1'='1' --
Senha: qualquer

Isso injeta SQL e bypassa o login!
SQL gerado pela aplicação:
// ❌ Código vulnerável (concatenação direta)
$sql = "SELECT * FROM usuarios_login
    WHERE usuario = '$usuario'
    AND senha = '$senha'";
$result = $conn->query($sql);

✅ Sistema Seguro

COM Prepared Statements
✅ Tente o mesmo ataque: Usuário: ' OR '1'='1' --
Senha: qualquer

O Prepared Statement bloqueará o ataque!
// ✅ Código seguro (Prepared Statement)
$stmt = $conn->prepare(
  "SELECT * FROM usuarios_login
   WHERE usuario = ? AND senha = ?"
);
$stmt->bind_param("ss", $usuario, $senha);
$stmt->execute();
